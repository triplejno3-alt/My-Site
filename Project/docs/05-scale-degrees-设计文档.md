Anki音乐学习模板设计文档
文档不许改动，除非经过批准。

## 1. 模块目标

- 训练用户快速将五线谱音符转换为键盘/音名操作（视奏基础）
- 建立从视觉（五线谱）到动作（键盘/按钮）的快速反射
- 通过乐句练习建立音程感和旋律记忆

## 2. 设计思路
    系统生成一段随机乐句（4-12个音符），用户需要在虚拟键盘上按顺序准确输入对应的音名。

    特点：
    - 实时反馈：输入正确后音符变绿，并播放对应音高。
    - 进度追踪：记录准确率和平均反应时间。
    - 难度自适应：根据表现自动提示或调整建议。

## 3. 技术实现
    - 使用 HTML/JS 创建交互式视奏界面。
    - 使用 VexFlow 库渲染动态五线谱，支持实时颜色更新。
    - 使用 Web Audio API 进行音频播放，支持乐句完整试听。
    - 使用 CSS 实现平滑的 UI 动画和自适应布局。

## 4. 答题反馈机制
    ## 4.1 正确反馈
    - 对应按钮高亮（紫色）
    - 五线谱对应音符变绿
    - 播放对应音高的音频
    - 下方输入序列填充对应字母

    ## 4.2 错误反馈
    - 对应按钮变红抖动
    - 记录错误次数，影响最终准确率

## 5. 功能特性

### 5.1 乐句系统
- **动态生成**：支持 4, 6, 8, 12 不同长度的乐句。
- **智能模式**：支持纯随机模式和级进（Step）优先模式，使旋律更具歌唱性。
- **范围选择**：
  - C4-G4（基础）
  - C4-C5（扩展）
  - A3-E5（完整）

### 5.2 实时数据看板
- **准确率**：(正确点击 / 总尝试次数) × 100%。
- **进度条**：动态显示当前准确率，颜色根据区间变化（红/橙/绿）。
- **完成结算**：显示总耗时和“平均每音耗时”。

### 5.3 学习数据分析（继承自T4）
- **趋势图表**：展示最近 20 次练习的平均用时趋势。
- **准确率分布**：展示准确率的波动情况。
- **散点分析**：分析用时与准确率的相关性。
- **历史记录**：保留最近练习的详细数据。

### 5.4 训练设置
- 可调节乐句长度。
- 可调节音域范围。
- 物理键盘支持（C-B键直接输入）。

## 6. 文件结构

### 6.1 多文件版本
- `index.html`：主结构文件
- `style.css`：交互样式文件
- `app.js`：核心逻辑与渲染引擎
- `audio/`：钢琴音源目录

## 7. 使用说明

### 7.1 操作流程
1. 点击“开始挑战”生成乐句。
2. 观察五线谱，按顺序在下方 C-B 按钮或物理键盘输入对应音名。
3. 输入正确会听到声音并看到音符变色。
4. 完成整段后查看总结评价。

## 8. 技术依赖
- **VexFlow库**：`https://unpkg.com/vexflow@4.0.1/build/cjs/vexflow-debug.js`
- **Web Audio API**：浏览器原生支持。
