Anki音乐学习模板设计文档 - T6 节奏练习
文档不许改动，除非经过批准。

## 1. 模块目标

- 建立精确的节奏感和时值概念
- 训练用户识别并重现基本节奏型（全音符、二分、四分、八分、十六分音符等）
- 提高在固定 BPM（节拍器）下的节奏稳定性
- 建立从节奏符号到动作执行的快速反射

## 2. 设计思路

系统展示一个乐句的节奏（通常 2-4 小节），提供节拍器引导，用户通过点击屏幕或空格键来“敲击”出对应的节奏。

特点：
- **实时对比**：在时间轴上实时显示标准节奏点与用户点击点的偏差。
- **阶梯难度**：从基础的整拍节奏逐步过渡到复杂的切分音和附点节奏。
- **视觉映射**：使用五线谱（打击乐谱表）渲染节奏符号，直观展示时值长短。

## 3. 技术实现

- **渲染引擎**：使用 VexFlow 渲染节奏谱（Rhythm Notation）。
- **时间检测**：利用 `performance.now()` 实现高精度的点击时间戳采集。
- **节奏判定**：计算点击时间点与理论时间点的毫秒差，根据误差范围判定等级（Perfect/Great/Good/Miss）。
- **音频控制**：使用 Web Audio API 实现稳定的节拍器脉冲和打击音效。

## 4. 训练流程

### 4.1 准备阶段
- 显示当前节奏谱。
- 设定 BPM（默认 80 bpm）。
- 用户点击“开始”进入预备拍。

### 4.2 执行阶段
- 节拍器播放预备拍（通常 4 拍）。
- 用户跟随节拍器敲击目标节奏。
- 屏幕实时反馈每次点击的准确度。

### 4.3 结算阶段
- 显示总准确率评分。
- 展示偏差散点图（时间偏差分析）。
- 提供历史进步曲线。

## 5. 功能特性

### 5.1 节奏型库
- **Level 1**：全音符、二分音符、四分音符组合。
- **Level 2**：加入八分音符（两个八分音符一组）。
- **Level 3**：加入四分附点及切分音基础。
- **Level 4**：十六分音符及其组合。
- **Level 5**：复杂附点、三连音等。

### 5.2 核心参数
- **BPM 调节**：支持 40 - 180 BPM。
- **预备拍开关**：可选择是否开启预备拍。
- **自动难度**：连续 Perfect 两次自动解锁下一关。

### 5.3 数据看板（继承自 T4）
- **节奏稳定性**：计算点击偏差的标准差。
- **误差分布图**：展示点击是偏早还是偏晚。
- **历史数据追踪**。

## 6. 文件结构

- `index.html`：布局文件。
- `style.css`：交互样式与时间轴动画。
- `app.js`：核心节奏引擎（Scheduler, Detector）。
- `audio/`：包含节拍器重音/轻音，以及打击反馈音。

## 7. 技术依赖

- **VexFlow**：用于节奏谱渲染。
- **Web Audio API**：用于高精度计时和音频合成。
