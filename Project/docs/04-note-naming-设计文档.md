Anki音乐学习模板设计文档
文档不许改动，除非经过批准。

## 1. 模块目标

- 训练用户快速将音名（CDEFGAB）转换为唱名数字（1234567）
- 建立音名、数字、唱名之间的快速条件反射
- 通过渐进式训练提高转换速度和准确性

## 2. 设计思路
    显示音名（如C），用户需要在选项中选择对应的数字（1）或唱名（Do）
    卡片中间显示音高，用户需要在选项中选择正确的音高名称（如C、D、E、F、G等）。通过反复练习，用户可以提高对不同音高的识别速度和准确性。

    特点：三重映射训练（音名↔数字↔唱名）

## 3. 技术实现
    - 使用HTML和JavaScript创建交互式选择界面。
    - 预定义选择按钮，用户点击后记录答案并提供反馈。
    - 使用CSS进行样式设计，确保界面美观且易于使用。
    - 使用vexflow库进行谱号的渲染和显示。
## 4. 答题反馈机制（核心更新）
    ## 4.1 正确反馈

    - 按钮变绿色（高亮）
    - 进入下一题

    ## 4.2 错误反馈
    当用户点击错误选项：
    - 按钮变红（错误提示）
    - 0.2 秒后自动变为灰色禁用状态

## 5. 功能特性（最新更新）

### 5.1 三重映射系统
- **音名到数字**：C → 1, D → 2, E → 3, F → 4, G → 5, A → 6, B → 7
- **音名到唱名**：C → Do, D → Re, E → Mi, F → Fa, G → Sol, A → La, B → Si
- **数字到音名**：1 → C, 2 → D, 3 → E, 4 → F, 5 → G, 6 → A, 7 → B

### 5.2 训练模式
- **模式1：基础转换** - 音名 → 数字
- **模式2：唱名转换** - 音名 → 唱名
- **模式3：反向转换** - 数字 → 音名
- **模式4：混合训练** - 随机题目类型

### 5.3 难度等级
- **等级1**：C、D、E三个音（基础训练）
- **等级2**：C、D、E、F、G五个音（扩展训练）
- **等级3**：全部七个音（完整训练）
- **等级4**：随机模式（综合测试）

### 5.4 实时正确率统计
- 每次答题后实时计算并显示正确率
- 正确率 = (正确数量 / 总数量) × 100%
- 使用进度条直观显示学习进度

### 5.5 通过机制
- 只有答题超过5题后才检查是否通过
- 当正确率超过90%时显示"恭喜！正确率超过90%，已通过！"
- 通过状态用绿色文字突出显示

### 5.6 进度条颜色系统
- **红色渐变**（≤70%）：表示需要加强学习
- **橙色渐变**（71%-90%）：表示进步中，接近目标
- **绿色渐变**（>90%）：表示已通过

### 5.7 进度条显示控制
- 答题数量≤5题时：进度条隐藏
- 答题数量>5题时：进度条显示
- 避免早期答题时的进度干扰

### 5.8 按钮布局优化
- 选项按钮水平单行排列
- 根据当前模式显示不同选项（数字1-7或字母C-B）

### 5.9 计分逻辑
- 正确答题：正确数量+1，总数量+1
- 错误答题：总数量+1
- 分数持续累积，不会重置

### 5.10 智能调度算法（继承T3）
- 基于正确率自动调整难度等级
- 错误频率统计，避免重复错误
- 冷却机制防止频繁升降级
- 渐进式学习曲线设计

## 7. 使用说明

### 7.1 多文件版本使用
1. 将所有文件放在同一目录下
2. 确保audio目录包含所有音频文件
3. 在浏览器中打开 `index.html`

### 7.2 训练流程
1. 选择训练模式（默认：音名→数字）
2. 系统显示一个音名（如"C"）
3. 从选项中选择对应的数字（"1"）
4. 系统提供即时反馈
5. 根据正确率自动调整难度
6. 达到90%正确率后进入下一等级
